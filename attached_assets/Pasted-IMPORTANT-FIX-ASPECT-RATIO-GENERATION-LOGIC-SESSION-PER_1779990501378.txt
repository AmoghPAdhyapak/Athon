ASPECT RATIO GENERATION ISSUE
=============================

Current issue:
When users upload a 9:16 image and request a 16:9 YouTube thumbnail, the AI still generates output in 9:16 ratio because the system is preserving the original uploaded image canvas size.

This must be fixed at the BACKEND GENERATION PIPELINE level, NOT only through prompt instructions.

==================================================
REQUIRED SOLUTION
=================

Create a proper Aspect Ratio Engine.

The system must:

* detect the selected target platform
* automatically create the correct output canvas dimensions
* intelligently recompose the uploaded image into the new ratio
* preserve the main subject visibility
* preserve face visibility
* preserve weapon visibility
* preserve storytelling clarity
* preserve the original visual identity of the uploaded image

DO NOT simply:

* stretch the image
* crop the image badly
* preserve original uploaded dimensions blindly
* redesign the uploaded image completely
